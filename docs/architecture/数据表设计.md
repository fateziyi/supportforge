# 核心数据表设计

> 最小业务闭环所需的 9 张核心表，所有表均遵循多租户隔离原则。

---

## 1.1 tenants（租户表）

存储平台入驻企业客户。

```sql
CREATE TABLE tenants (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name        VARCHAR(255) NOT NULL UNIQUE,
    status      VARCHAR(32)  NOT NULL DEFAULT 'active',
    plan        VARCHAR(32)  DEFAULT 'basic',
    created_at  TIMESTAMP    NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMP    NOT NULL DEFAULT NOW()
);
```

---

## 1.2 users（用户表）

租户下管理员、客服账号。

```sql
CREATE TABLE users (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id     UUID NOT NULL REFERENCES tenants(id),
    username      VARCHAR(100) NOT NULL,
    email         VARCHAR(255) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    role          VARCHAR(32)  NOT NULL,  -- platform_admin / tenant_admin / support_agent / auditor
    status        VARCHAR(32)  NOT NULL DEFAULT 'active',
    created_at    TIMESTAMP    NOT NULL DEFAULT NOW(),
    updated_at    TIMESTAMP    NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_users_tenant_id ON users(tenant_id);
```

---

## 1.3 knowledge_bases（知识库表）

```sql
CREATE TABLE knowledge_bases (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id   UUID NOT NULL REFERENCES tenants(id),
    name        VARCHAR(255) NOT NULL,
    description TEXT,
    status      VARCHAR(32) NOT NULL DEFAULT 'active',
    created_at  TIMESTAMP    NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMP    NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_knowledge_bases_tenant_id ON knowledge_bases(tenant_id);
```

---

## 1.4 documents（文档表）

```sql
CREATE TABLE documents (
    id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id         UUID NOT NULL REFERENCES tenants(id),
    knowledge_base_id UUID NOT NULL REFERENCES knowledge_bases(id),
    filename          VARCHAR(255) NOT NULL,
    file_path         TEXT NOT NULL,
    file_type         VARCHAR(32) NOT NULL,
    status            VARCHAR(32) NOT NULL DEFAULT 'uploaded',
    chunk_count       INT NOT NULL DEFAULT 0,
    vector_count      INT NOT NULL DEFAULT 0,
    created_at        TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at        TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_documents_tenant_id ON documents(tenant_id);
CREATE INDEX idx_documents_knowledge_base_id ON documents(knowledge_base_id);
```

---

## 1.5 conversations（对话表）

```sql
CREATE TABLE conversations (
    id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id         UUID NOT NULL REFERENCES tenants(id),
    user_id           UUID NOT NULL REFERENCES users(id),
    knowledge_base_id UUID REFERENCES knowledge_bases(id),
    title             VARCHAR(255),
    status            VARCHAR(32) NOT NULL DEFAULT 'open',
    last_message_at   TIMESTAMP,
    created_at        TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at        TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_conversations_tenant_id ON conversations(tenant_id);
CREATE INDEX idx_conversations_user_id ON conversations(user_id);
```

---

## 1.6 conversation_messages（对话消息表）

```sql
CREATE TABLE conversation_messages (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id       UUID NOT NULL REFERENCES tenants(id),
    conversation_id UUID NOT NULL REFERENCES conversations(id),
    role            VARCHAR(32) NOT NULL,  -- user / assistant / system
    content         TEXT NOT NULL,
    sources         JSONB,
    token_usage     JSONB,
    latency_ms      INT,
    created_at      TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_messages_tenant_id ON conversation_messages(tenant_id);
CREATE INDEX idx_messages_conversation_id ON conversation_messages(conversation_id);
```

---

## 1.7 tickets（工单表）

```sql
CREATE TABLE tickets (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id       UUID NOT NULL REFERENCES tenants(id),
    conversation_id UUID REFERENCES conversations(id),
    subject         VARCHAR(255) NOT NULL,
    description     TEXT NOT NULL,
    status          VARCHAR(32) NOT NULL DEFAULT 'open',
    priority        VARCHAR(32) NOT NULL DEFAULT 'medium',
    source          VARCHAR(32) NOT NULL DEFAULT 'manual',  -- manual / agent
    assignee_id     UUID REFERENCES users(id),
    created_at      TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_tickets_tenant_id ON tickets(tenant_id);
CREATE INDEX idx_tickets_assignee_id ON tickets(assignee_id);
```

---

## 1.8 audit_logs（审计日志表）

```sql
CREATE TABLE audit_logs (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id       UUID NOT NULL REFERENCES tenants(id),
    user_id         UUID REFERENCES users(id),
    action          VARCHAR(64) NOT NULL,
    resource_type   VARCHAR(64),
    resource_id     UUID,
    request_payload JSONB,
    response_payload JSONB,
    latency_ms      INT,
    created_at      TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_audit_logs_tenant_id ON audit_logs(tenant_id);
CREATE INDEX idx_audit_logs_user_id ON audit_logs(user_id);
```

---

## 1.9 agent_runs（Agent 执行记录表）

```sql
CREATE TABLE agent_runs (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id       UUID NOT NULL REFERENCES tenants(id),
    conversation_id UUID REFERENCES conversations(id),
    user_id         UUID REFERENCES users(id),
    status          VARCHAR(32) NOT NULL,
    category        VARCHAR(64),
    ticket_created  BOOLEAN NOT NULL DEFAULT FALSE,
    input_text      TEXT NOT NULL,
    output_text     TEXT,
    created_at      TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_agent_runs_tenant_id ON agent_runs(tenant_id);
```

## 前端核心页面

| 页面 | 路径 | 核心功能 |
|------|------|----------|
| 登录页 | `/login` | 邮箱密码登录、获取 Token |
| 仪表盘 | `/dashboard` | 今日对话 / 工单 / 文档统计、AI 耗时、命中率 |
| 知识库列表 | `/knowledge-bases` | 创建 / 编辑 / 删除知识库 |
| 文档管理 | `/documents` | 上传、解析状态、重解析、删除 |
| 对话调试 | `/conversations` | 客服对话、来源展示、工单转流 |
| 工单管理 | `/tickets` | 工单列表、状态筛选、指派、详情 |
| 审计日志 | `/audit-logs` | 操作记录、AI 调用、请求响应 |
| 用户权限 | `/settings/users` | 用户管理、角色分配、启停账号 |
| 系统设置 | `/settings` | 租户信息、API / 模型配置 |
