# 项目结构与文件职责

---

## 一、仓库级结构

```
SupportForge/
├── frontend/           # 前端 Next.js 项目
├── backend/            # FastAPI 后端项目
├── docs/               # 项目文档
│   ├── api/            # 接口文档
│   └── architecture/   # 架构文档
├── deploy/             # 部署配置、Docker 脚本
├── scripts/            # 自动化脚本
├── docker-compose.yml  # 一键部署
├── CLAUDE.md           # 开发规范
└── README.md           # 项目说明
```

---

## 二、后端文件职责说明

### 入口与全局配置 (`backend/app/core/`)

| 文件 | 职责 |
|------|------|
| `main.py` | FastAPI 应用入口，注册中间件、挂载路由、启动时初始化资源 |
| `core/config.py` | 统一管理配置项（数据库 URL、Redis URL、Qdrant URL、JWT 密钥、LLM API Key、环境名称） |
| `core/logging.py` | 初始化日志系统，统一日志格式，支持结构化日志输出（request_id 注入、error/info/audit log） |
| `core/exceptions.py` | 定义项目统一异常（BusinessException、UnauthorizedException、ForbiddenException），全局异常处理器 |
| `core/security.py` | JWT 生成与解析、密码哈希与验证（create_access_token、create_refresh_token、verify_password、hash_password） |
| `core/context.py` | 保存请求级上下文，管理当前 tenant_id、user_id（ContextVar），方便服务层/仓储层自动读取 |

### 数据库 (`backend/app/db/`)

| 文件 | 职责 |
|------|------|
| `db/base.py` | 定义 ORM 基类（Base = DeclarativeBase），公共 mixin，时间戳字段基类 |
| `db/session.py` | 创建数据库连接，提供 Session 依赖（async_engine、async_sessionmaker、get_db()） |
| `db/init_db.py` | 初始化数据库，创建初始数据（默认租户/管理员账号、demo 数据） |
| `db/migrations/` | Alembic 数据库迁移脚本，表结构版本管理 |

### 模型层 (`backend/app/models/`)

| 文件 | 职责 |
|------|------|
| `models/tenant.py` | 定义租户表（id、name、status、created_at、updated_at） |
| `models/user.py` | 定义用户表（id、tenant_id、username、email、password_hash、role） |
| `models/knowledge_base.py` | 定义知识库表（id、tenant_id、name、description） |
| `models/document.py` | 定义文档表（id、tenant_id、knowledge_base_id、filename、status、file_path） |
| `models/conversation.py` | 定义对话表（id、tenant_id、user_id、status、last_message_at） |
| `models/ticket.py` | 定义工单表（id、tenant_id、conversation_id、subject、status、priority、assignee_id） |
| `models/audit_log.py` | 定义审计日志表（id、tenant_id、user_id、action、resource_type、resource_id、payload、created_at） |

### Schema 层 (`backend/app/schemas/`)

| 文件 | 职责 |
|------|------|
| `schemas/auth.py` | 登录、刷新 token、返回用户信息的请求响应模型 |
| `schemas/tenant.py` | 租户创建、更新、返回结构 |
| `schemas/user.py` | 用户创建、更新、返回结构（含角色字段） |
| `schemas/knowledge_base.py` | 知识库创建、更新、详情返回结构 |
| `schemas/document.py` | 文档上传、列表、详情、状态返回结构 |
| `schemas/conversation.py` | 对话创建、消息发送、消息返回结构 |
| `schemas/ticket.py` | 工单创建、更新、详情返回结构 |
| `schemas/audit_log.py` | 审计日志查询与返回结构 |

### API 层 (`backend/app/api/`)

| 文件 | 职责 |
|------|------|
| `api/deps.py` | FastAPI 依赖（获取当前用户、获取当前租户、权限校验、数据库会话） |
| `api/v1/router.py` | 聚合所有 v1 路由，统一挂载前缀 `/api/v1` |
| `api/v1/auth.py` | 登录、刷新 token、当前用户信息、退出登录接口 |
| `api/v1/tenants.py` | 租户创建、查询、更新、启停接口 |
| `api/v1/users.py` | 用户 CRUD、角色管理接口 |
| `api/v1/knowledge_bases.py` | 知识库 CRUD、统计接口 |
| `api/v1/documents.py` | 文档上传、列表、详情、重解析、状态查询接口 |
| `api/v1/conversations.py` | 创建对话、发送消息、获取消息记录、关闭对话接口 |
| `api/v1/tickets.py` | 工单创建、列表、更新状态、指派、关闭、备注接口 |
| `api/v1/audit_logs.py` | 审计日志查询、详情、汇总统计接口 |

### 服务层 (`backend/app/services/`)

| 文件 | 职责 |
|------|------|
| `services/auth_service.py` | 登录认证业务、用户校验、token 签发、刷新逻辑 |
| `services/tenant_service.py` | 租户创建与管理、租户启停、租户配置读取 |
| `services/knowledge_service.py` | 知识库业务处理、文档归属知识库关系管理 |
| `services/document_service.py` | 文档上传后的业务处理、调度解析任务、更新文档状态 |
| `services/ticket_service.py` | 工单创建、更新、流转逻辑，Agent 转工单后的业务承接 |
| `services/conversation_service.py` | 对话上下文管理、消息保存、调用 Agent 并处理结果 |
| `services/audit_service.py` | 审计日志写入、关键行为追踪、AI 调用记录 |
| `services/rag_service.py` | 封装 RAG 主流程（检索、重排序、回答生成） |

### 仓储层 (`backend/app/repositories/`)

| 文件 | 职责 |
|------|------|
| `repositories/base.py` | 仓储基类，通用 CRUD 方法，统一 tenant 过滤模式 |
| `repositories/tenant_repo.py` | 租户表的数据访问 |
| `repositories/user_repo.py` | 用户表数据访问，用户按租户查询 |
| `repositories/knowledge_base_repo.py` | 知识库数据访问 |
| `repositories/document_repo.py` | 文档数据访问，文档状态更新 |
| `repositories/ticket_repo.py` | 工单数据访问（状态、指派、关闭查询） |
| `repositories/conversation_repo.py` | 对话与消息数据访问 |
| `repositories/audit_log_repo.py` | 审计日志查询与写入 |

### Agent 层 (`backend/app/agent/`)

| 文件 | 职责 |
|------|------|
| `agent/graph.py` | 定义 LangGraph 工作流，组织节点、边、状态流转 |
| `agent/states.py` | 定义 Agent 共享状态结构（分类结果、检索结果、回复结果、工单信息） |
| `agent/nodes/classifier.py` | 问题分类节点（判断是简单回答还是需要转工单） |
| `agent/nodes/retriever.py` | 检索知识库节点（调用 RAG 检索逻辑） |
| `agent/nodes/responder.py` | 生成回复建议节点（结合上下文与检索结果输出答案） |
| `agent/nodes/ticket_creator.py` | 复杂问题转工单节点（组织工单创建信息） |
| `agent/nodes/human_in_loop.py` | 人工接管节点（需要客服确认时暂停或转人工） |
| `agent/tools/ticket_tools.py` | Agent 工单相关工具函数（创建工单、查工单状态） |
| `agent/tools/knowledge_tools.py` | Agent 知识库检索工具 |
| `agent/tools/audit_tools.py` | Agent 审计记录工具 |
| `agent/prompts/classify.md` | 分类提示词模板 |
| `agent/prompts/respond.md` | 回复生成提示词模板 |
| `agent/prompts/escalate.md` | 转工单提示词模板 |

### RAG 层 (`backend/app/rag/`)

| 文件 | 职责 |
|------|------|
| `rag/chunking.py` | 文档分段（按长度、标题、语义切块） |
| `rag/embedding.py` | 文本向量化（封装 embedding 模型调用） |
| `rag/retriever.py` | Qdrant 检索（tenant 过滤，返回 top-k 结果） |
| `rag/rerank.py` | 检索结果重排序（提高答案相关性） |
| `rag/ingestion.py` | 文档入库主流程（chunk → embedding → upsert） |
| `rag/evaluators.py` | Ragas 评估（检索质量、回答质量指标计算） |

### 任务层 (`backend/app/tasks/`)

| 文件 | 职责 |
|------|------|
| `tasks/celery_app.py` | Celery 应用初始化，连接 Redis 作为 broker/backend |
| `tasks/document_tasks.py` | 文档解析异步任务，文档切分与状态更新 |
| `tasks/embedding_tasks.py` | 文本向量化和入库异步任务 |
| `tasks/ticket_tasks.py` | 工单相关异步任务（通知、同步、自动分配） |
| `tasks/cleanup_tasks.py` | 定期清理任务（过期缓存、失败任务、临时文件清理） |

### 集成层 (`backend/app/integrations/`)

| 文件 | 职责 |
|------|------|
| `integrations/qdrant_client.py` | 封装 Qdrant 客户端初始化与通用方法 |
| `integrations/redis_client.py` | 封装 Redis 连接 |
| `integrations/jira_client.py` | 封装 Jira 工单系统集成（复杂问题自动创建开发任务） |
| `integrations/tracing.py` | LangSmith / OpenAI Tracing 集成（Agent 执行链路追踪） |

### 工具层 (`backend/app/utils/`)

| 文件 | 职责 |
|------|------|
| `utils/pagination.py` | 分页参数与分页结果封装 |
| `utils/ids.py` | UUID / ID 生成辅助 |
| `utils/time.py` | 时间格式化、时区处理 |
| `utils/helpers.py` | 通用辅助函数（字符串处理、字典合并等） |

### 测试目录 (`backend/tests/`)

| 目录 | 职责 |
|------|------|
| `tests/unit/` | 单元测试（service、工具函数、状态流转） |
| `tests/integration/` | 集成测试（数据库、Qdrant、Redis、API 联调） |
| `tests/e2e/` | 端到端测试（完整业务闭环） |

## 四、推荐开发顺序

```
1. core / db / models     → 基础架构
2. schemas / repositories / services → 业务逻辑
3. api                    → 接口路由
4. rag / agent / tasks    → AI 能力
5. frontend + tests       → 前端页面 & 测试
```
