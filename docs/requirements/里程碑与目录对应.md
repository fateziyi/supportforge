## 开发策略建议：按"功能域"推进

> 不要按技术分散写，而是按业务域逐域闭环。

| 域 | 逐域闭环路径 |
|----|-------------|
| `knowledge_base` | model → schema → repository → service → API → 前端页面 |
| `document` | model → schema → repository → service → API → 前端页面 |
| `conversation` | model → schema → repository → service → API → 前端页面 |
| `ticket` | model → schema → repository → service → API → 前端页面 |
| `audit` | model → schema → repository → service → API → 前端页面 |
